Что уже есть, что нужно сделать

Инвентаризация перед брейнстормом — кликни блоки которые хочешь включить в скоп

Zero Block Editor

✅ УЖЕ РАБОТАЕТ
  • Snap-to-grid + snap-to-elements (snap-engine.ts)
  • 6 брейкпоинтов (desktop, 1200, 980, 640, 480, 320)
  • Responsive overrides per element per breakpoint
  • Все 10 типов элементов в панели «Добавить»
  • Рендер tooltip, form, gallery на холсте
  • Undo/redo, copy/paste, z-order, alignment
  • Настройки для text, image, shape, button, vector, video, html
❌ ОТСУТСТВУЕТ
  • Панели настроек для tooltip, form, gallery (returns null)
  • Breakpoint panel: нет 480px и 320px в top bar
  • Snap threshold и gridSize не настраиваются из UI
⚠️ ЕСТЬ НО МОЖНО УЛУЧШИТЬ
Snap: нет snap-to-center по canvas-wide Resize snap только на grid, не на соседей Нет дистрибуции элементов (distribute)

GrapesJS / Lemnity Box Editor

✅ УЖЕ РАБОТАЕТ
  • Tilda-style insert zones между секциями
  • Toolbar: sibling moves (↑↓), block settings, save block
  • Section width grid overlay
  • Device switcher (desktop/tablet/phone)
  • Style manager dropdowns (замена нативных)
  • Right panels overlay (collapse/expand)
  • Block library flyout (левый сайдбар)
  • User blocks panel (новый)
  • Canvas viewport guides
🔧 МОЖНО ДОБАВИТЬ
  • Inline double-click editing текста
  • Quick color picker в тулбаре секции
  • Section background: цвет/градиент/фото прямо из тулбара
  • Paste HTML/секции из буфера (Ctrl+Shift+V)
  • Счётчик секций + быстрая навигация к секции
  • Keyboard shortcut: Delete = удалить секцию
Вопрос: Какой приоритет правильный? Сначала довести Zero Block до полной готовности (настройки tooltip/form/gallery), или сфокусироваться на GrapesJS UX улучшениях, или оба пускаем вместе в одном плане?
