Изменения модели данных

Минимальные расширения существующего SlideGraph — без ломающих изменений

SlideElement — добавляем 3 поля
export interface SlideElement {
id: string
type: SlideElementType
frame?: SlideElementFrame
// ... существующие поля

// ← новые поля для редактора
name?: string
// "Заголовок", "Логотип" — для панели слоёв
locked?: boolean
// 🔒 заблокирован от drag/resize
visible?: boolean
// 👁 скрыт в редакторе (default: true)
}
✅ Обратно совместимо
Старые SlideGraph без этих полей работают без изменений — undefined читается как незаблокировано/видимо
SlideTheme — дизайн-токены
export interface SlideTheme {
// существующие
primaryColor: string
accentColor?: string
backgroundColor: string
textColor: string
fontFamily: string

// ← новые токены
headingFont?: string
secondaryColor?: string
surfaceColor?: string
// цвет карточек/панелей
borderRadius?: 'none'|'sm'|'md'|'lg'
spacing?: 'compact'|'normal'|'spacious'
}
Все новые поля опциональны
buildSlideDeckStyles() получает дефолты, если поля отсутствуют
Slide — добавляем transition (для анимаций, фаза 3)
export interface Slide {
id: string
layout: SlideLayout
freeform?: boolean
background?: SlideBackground
elements: SlideElement[]
notes?: string

// ← фаза 3 (сейчас просто тип)
transition?: SlideTransition
}

export interface SlideTransition {
type: 'none'|'fade'|'slide'|'zoom'
duration?: number // ms
direction?: 'left'|'right'|'up'|'down'
}
Что НЕ меняем
• SlideGraph.version остаётся 1
• SlideLayout и SlideElementType — без изменений
• API-маршруты /api/projects/[id]/slides/* — совместимы
• Существующие PPTX/PDF экспорты — работают
Итог изменений
• +3 поля в SlideElement
• +5 опциональных токенов в SlideTheme
• +1 опциональное поле transition в Slide
• Нулевая миграция данных
