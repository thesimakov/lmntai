Архитектура компонентов — AI-редактор

Как build/page.tsx разбивается на части. Зелёное — новое, синее — рефактор существующего, серое — без изменений.

Файловая структура (новая)
app/(builder)/playground/build/
page.tsx ← тонкий оркестратор (auth + routing + <AiEditorShell>), ~150 строк
components/ai-editor/
AiEditorShell.tsx ← новый корневой layout (левый сайдбар + центральный превью)
AiEditorSidebar.tsx ← список версий + ввод промпта + история чата
AiVersionList.tsx ← прокручиваемый список снимков с датой и промптом
AiVersionDiffBadge.tsx ← маленький индикатор «v3 · 2 мин назад»
AiPromptInput.tsx ← вынесен из build/page.tsx (textarea + кнопка + модель)
AiEditorPreview.tsx ← обёртка над preview-frame.tsx + тулбар устройств
components/playground/
preview-frame.tsx ← добавляем экшены в ElementEditorPanel (AI-правка)
agent-chat.tsx ← без изменений (используется внутри AiEditorSidebar)
lib/stores/
use-build-editor-store.ts ← добавляем: currentVersionId, versions[], selectedElementId
app/api/projects/[id]/snapshots/
route.ts ← GET (список), POST (создать снимок)
[snapshotId]/route.ts ← GET (загрузить), DELETE
prisma/migrations/
add_project_snapshot ← новая таблица ProjectSnapshot
Дерево компонентов (runtime)
build/page.tsx
└─ AiEditorShell
├─ AiEditorSidebar (260px, фикс.)
├─ AiVersionList → AiVersionDiffBadge ×N
├─ AgentChat (существующий)
└─ AiPromptInput
└─ AiEditorPreview (flex-1)
├─ BuildPreviewChrome (топбар)
└─ PreviewFrame (+ inline toolbar)
Принцип разбивки: build/page.tsx сохраняет всю логику потоков (SSE, Lemnity AI bridge, токены) — но рендерит только <AiEditorShell>. UI полностью вынесен. Это позволяет изменять раскладку без прикосновения к логике стриминга.
