Карта подпроектов Lemnity

5 независимых направлений — каждое требует своего spec + плана. С чего начнём?

🤖

1. AI-редактор (UX)

Переработка build-редактора под lovable.dev / v0.app: левый сайдбар с историей чата, главная область — интерактивный превью с оверлеем для выбора компонентов, история версий.

~build/page.tsx уже 2500 строк
🧱

2. Box — Пользовательские блоки

«Сохранить как блок» в библиотеку, вкладка «Мои блоки» в flyout, хранение в БД (UserVirtualWorkspace или новая модель), CRUD API.

Не реализовано
⬡

3. Zero Block — доработки

Расширение типов элементов (видео, иконки, фигуры), улучшение тулбара, snap-to-grid, адаптивные breakpoints внутри Zero Block.

Базовая версия есть
🔄

4. Совместимость редакторов

Единая JSON-модель (PageDocument), конвертер AI-HTML → Box-блоки и обратно, синхронизация черновика между режимами без потери данных.

Схема есть, конвертер — нет
🎨

5. Дизайн-система

Единая стилистика обоих редакторов: современный минимализм, палитра (нейтралы + 1 акцент), строгая типографика. theme-factory + brand-guidelines.

Финальный этап

💡 Моя рекомендация: начать с #2 — Пользовательские блоки

Самая самодостаточная задача, не зависит от остальных, даёт сразу видимую ценность пользователям Box-редактора. #4 (совместимость) — наиболее рискованная с архитектурной точки зрения, лучше сделать её после того, как оба редактора устойчивы.
